## Known Gap: Merchant Start Auction Button

The backend already supports starting an auction:

```http
POST /api/v1/auctions/:id/activate
```

But the merchant frontend currently lacks a visible "开拍" button.

This is a good fast-lane task because it wires one existing backend action into the UI and should not require a new OpenSpec change unless scope expands.

Likely implementation:

- Add `activateAuction(auctionId: number): Promise<void>` in `frontend/src/api/auction.ts` or another consistent API module.
- In `frontend/src/pages/merchant/ProductDetail.tsx`, when `auction` exists and `product.status === 'pending'` or `auction.status === 'pending'`, show `开拍`.
- On click, call `activateAuction(auction.id)`, then refresh detail or navigate back to product detail.
- Show visible error if activation fails.
- Focused verification:
  - `cd frontend && npm run build`
  - targeted ESLint for touched files
  - browser smoke or Playwright/component test if practical
