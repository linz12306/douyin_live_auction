- User asked to optimize WS concurrency after local `5000/500/300WS` queued load showed realtime `dropped_events` increased by about 310 while HTTP enqueue, worker consumption, Redis command group, and DB invariants stayed healthy.

- Updated OpenSpec/design/tasks and Superpowers plan under existing `auction-bid-command-stream`.
- Added RED tests:
  - `TestRedisStreamAuctionEventBusBuffersBurstySubscriberWithoutDrops` failed with 240 dropped events before implementation.
  - `TestAsyncBidCommandEnqueueDoesNotPublishQueuedRealtimeEvent` failed because enqueue emitted `bid_command.status` `queued`.
- Implemented:
  - Redis realtime subscriber buffer increased to 4096 for Redis Stream subscribers.
  - `EnqueueBidCommand` no longer publishes per-command queued realtime events; HTTP response and MySQL command row remain the queued source. Worker-owned processing/accepted/rejected/failed status events remain.
