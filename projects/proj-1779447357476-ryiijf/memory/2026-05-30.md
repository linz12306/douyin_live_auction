## Merchant Dashboard

- Implemented OpenSpec change `merchant-dashboard`.
- Added backend `GET /api/v1/merchant/dashboard` for merchant role users only.
- Dashboard response includes product status counts, order status counts, paid transaction summary, active auction summaries, and recent orders scoped to the current merchant.
- Transaction metrics intentionally count only `paid` orders: `total_paid_amount`, `paid_order_count`, and `average_paid_price`.
- Added frontend `/merchant/dashboard` with metric cards, product/order status sections, active auctions, recent orders, loading/error states, and links back to product/order management.
- Added dashboard entry links from merchant product list, merchant order list, and profile.

## Environment Notes

- This worktree uses Redis `127.0.0.1:16380` for backend and integration tests.
- Local default `auction_db` was being affected by an existing backend/server process and historical test data, producing foreign-key and missing-image flakes during full integration verification.
- To isolate verification, created local DB `auction_db_merchant_dashboard` from migrations `001` through `007`.
- Backend tests require UTC location in the MySQL DSN for reliable comparisons because MySQL `NOW()` and Go local time otherwise differ in this environment.
